Technical Blog 6 - JavaScript Language

Explain to a non-tech friend that in the JavaScript language, all the use cases for each of:

  • Parentheses (fancy way of saying round brackets...)() - e.g. 'Execute a function var result = doSomething()'

    • There are several different places you will see a () in JavaScript:

    1. Next to a function but empty e.g. function startGame()

      2. The () in this case simply tells the computer 'Hey! This is a function so you're gonna have to do stuff!' because without the () it's just a word and the computer is like: 'Uh-uh. And? What do you want me to DO?'

    2. Next to a function but with parameters in it E.g. function createBoard(size)

      3. Everything in between the parentheses (aka round brackets) are the 'sureyors' bit to a function. You are simply telling the computer 'Hey! We're gonna create a board and I'm gonna tell you what size it is.'. It usually contains a list of parameters (I think of them as boundary pegs). Some functions have no parameters so it's blank like above (and I'm still amazed that it still works!).

    3. Containing an expression or 'doing code' e.g. for (let i = 0; i < size; i++)

      4. This is like the actual receipe for doing things. In this example code, you saying to the computer 'Ok buddy, I want you to start at 0 and then size up by one each time.'

    4. Normal algebra e.g. a = (4 + 3) * 2

      5. Another way that parenthese is used is when you have an algebraic equation (remember high school maths? I know...ugh!). But it just means you do the equation inside first.

  • Brackets []

    • Inside the brackets [] is where you put a list (fancy name = 'array') in JavaScript.

  • Braces {}

    • You put code in curly braces to create objects (things that can be made) and declare variables (names for things). E.g. var board = { cells: ...}

  • Single quotes ''

    • You write your strings (words and sentences) in quotes. This tells the computer to recognise it as just words, not special names for a function.

  • Double quotes ""

    • Had to google this. There is no difference between single quotes '' and double quotes "" except that you can write a double quote in a single-quoted string without escaping it.
